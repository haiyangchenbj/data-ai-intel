📌 月度总判断

4 月数据平台行业的核心叙事是：从"谁是 AI Agent 的控制平面"分化出三条清晰路线——Snowflake 以平台内聚式"Agentic Enterprise 控制平面"整合业务用户与开发者双端，Google Cloud 以跨云"Agentic Data Cloud"直接挑战 Databricks/Snowflake 的数据驻留地位，Databricks 以治理自动化三连发（Governed Tags → 存储位置管理 → Data Classification）将 Unity Catalog 推向"自动化治理引擎"。三条路线之外，算力涨价潮从海外蔓延至国内（阿里云 4/18、百度 4/18、腾讯云 5/9 陆续生效），国家数据局将"数据基础设施"首次设为独立政策赛道，国内 Agent 治理进入标准化落地阶段。

5 月关注：Google Cloud 跨云 Lakehouse GA 时间表、Databricks Supervisor Agent SDK GA 时间线、华为云 AgentArts 公测（4/30）实际体验、06 月 Databricks Data+AI Summit / Snowflake Summit 预热发布。

趋势

月度趋势分析

趋势一：三条"Agent 控制平面"路线正式分化 ★★★（高可信）

论据：Snowflake（04-21）+ Google Cloud（04-24）+ Databricks（04-17/20/24）三大平台在同一月份分别以截然不同的路线公开推进"Agent 控制平面"定位，事件跨厂商、跨地域、路线互斥，结构性证据充分。

4 月最大结构性事件是三大平台在"Agent 控制平面"上的路线分歧从暗斗走向公开竞争。Snowflake 在 04-21 以"control plane for the Agentic Enterprise"定位同时推出 Intelligence 升级（个人工作代理+MCP 连接器+iOS App）和 Cortex Code 扩展（主动连接 AWS Glue/Databricks、PostgreSQL）——数据平台首次以"控制平面"定位覆盖业务用户和开发者双端，且主动连接竞品数据源。竞争逻辑从"让数据搬进来"转向"无论数据在哪，都通过我来编排 Agent"。

Google Cloud 在 04-24 Next '26 Keynote 发布 Agentic Data Cloud，包含跨云 Lakehouse（基于 Iceberg REST Catalog 零拷贝读取 Databricks/Snowflake 数据）、Knowledge Catalog（Gemini 自动富化业务上下文）和 Data Agent Kit，直接将 BigQuery 重新定位为"推理平面"——数据在哪不重要，推理和上下文层跑在 Google。

Databricks 则以"治理控制平面"路线推进——通过连续三项治理 GA（Governed Tags → 存储位置管理 → Data Classification）将 Unity Catalog 从元数据注册中心推向自动化治理引擎，并通过 Supervisor Agent SDK（Beta）将多 Agent 编排工程化。

So what：三条路线的核心争议是"谁来定义 Agent 的业务语义"。Snowflake 占业务用户入口，Google 占语义层，Databricks 占数据治理权。06 月双峰会将决定谁能拿到最多企业 Agent 采用。

趋势二：算力涨价潮从海外蔓延至国内，FinOps 从可选变为结构性需求 ★★★（高可信）

论据：阿里云（3/18 公告，4/18 生效，幅度 5%-34%）+ 百度智能云（同步跟进，5%-30%）+ 腾讯云（4/9 公告，5/9 生效，5%）+ 海外 AWS/Google Cloud 涨价已在先，加上国家数据局披露我国日均 Token 调用量 140 万亿的官方数字，五方事件独立可核。

阿里云、百度智能云、腾讯云集体上调 AI 算力价格，幅度 5%-34%，标志"算力普惠时代"正式终结。值得注意的是涨价并非一次性同日生效，而是阿里云 4/18、百度 4/18、腾讯云 5/9 的分步落地。国家数据局披露我国日均 Token 调用量已达 140 万亿（2024 年初仅 1000 亿），Token 正从技术计量单位加速转变为产业货币。

Futurum 分析师 Brad Shimmin 在 04-24 明确警告"智能锁定"风险：Google Knowledge Catalog 的语义映射深度绑定 Gemini 模型，"迁移大脑比迁移数据困难得多"。这与 IDC/Gartner 关于语义层的判断形成呼应——企业选型面临的双重锁定从"存储锁定"升级为"语义锁定+智能锁定"。

So what：Token 消耗监控、推理成本归因、智能模型路由将从"锦上添花"变成平台标配。国内数据要素×大赛首次将"算力智能感知与调度"设为独立赛题，政策层面已将算力治理纳入基础设施议题。

趋势三：数据平台治理从平台内建向跨平台互操作演进 ★★（中可信）

论据：两条独立事件：Google × Collibra 双向元数据集成 Preview（04-22）+ 信通院可信 Skills 治理首批测试（04-21）。前者跨厂商治理互操作，后者标准化治理落地。证据链有效但事件数相对少，未达到 ★★★ 阈值。

Google Cloud × Collibra 双向元数据集成（04-22 Preview）是标志性信号：Databricks（Unity Catalog）、Snowflake（Horizon）、Oracle（AIDP Workbench）此前均走"治理跟着平台走"的内建路线。Collibra 打通 Dataplex 双向通道，意味着第三条路线出现——"治理独立于任何单一平台"。

同时信通院启动可信 Skills 治理首批测试（04-21），3 大类 15 指标 70+ 细分项覆盖 Agent 技能执行可信性/调度可控/合规安全，7 月发布结果。标志着国内数据平台 Agent 治理合规即将进入标准化落地阶段。

So what：治理层的竞争焦点正在从"有没有治理"转向"治理能否覆盖 Agent 场景"以及"治理语义能否跨平台流动"。这个判断需要 5-6 月更多独立事件（如 Informatica/Alation/Atlan 的治理互操作公告）进一步印证。

趋势四：Iceberg V3 成为开放湖仓标准，标准之争收官 ★★★（高可信）

论据：Iceberg Summit 2026（04-08/09）+ Snowflake Iceberg V3 Preview + Databricks Iceberg V3 Preview + Google/Dremio 同步支持，四家湖仓引擎 + 一次行业峰会构成完整事件矩阵。

Iceberg Summit 2026（04-08/09）汇聚 700+ 人线下参会，四大湖仓引擎——Snowflake、Databricks、Google、Dremio——全面发布 Iceberg V3 支持。Databricks 在 Summit 后一周（04-16）发布 Iceberg V3 Preview on Databricks，核心新功能包括行级血缘（每行数据永久 ID + 修改时间戳）和 VARIANT 列类型。Snowflake 同步深化"数据自主权"路线，将 Iceberg V3 定为跨平台分析的运营基础而非仅仅是兼容格式。

So what：Iceberg 不再是"候选标准之一"，它就是标准本身。开放湖仓的竞争已从"是否支持 Iceberg"升级为"Iceberg 生态的深度和广度"。Databricks 开放原生写入（而非仅通过 Uniform）的能力是 06 月峰会的重要看点之一。

趋势五：AI 基础设施层资本虹吸加剧 ★★（中可信，较原版降级）

论据：Oracle 密歇根 $160 亿（04-24）+ VAST Data $10 亿（04-23）+ Ineffable Intelligence $11 亿（04-27）三条大额融资构成 AI 基础设施层证据。但"数据平台独立赛道被压缩"的对比判断缺少同量级反向证据（如 Databricks/Snowflake 同期未见大规模融资动作），因此从"两极分化"调整为"基础设施层虹吸"的单向描述。

Oracle 密歇根数据中心 $160 亿融资落地（Pimco $100 亿债券 + Blackstone/Related Digital 股权；1GW 服务 Stargate/OCI），VAST Data F 轮 $10 亿（估值 $300 亿，Drive Capital + Access Industries 领投），Ineffable Intelligence 种子轮 $11 亿（Sequoia + Lightspeed 领投，欧洲史上最大种子轮）。AI 基础设施层和前沿模型层的资本虹吸效应持续加剧。

So what：数据平台厂商需要关注的是——AI 基础设施赛道和模型赛道正在虹吸资本和 IPO 窗口，Databricks 的 S-1 节奏对整个数据平台赛道有指标意义。"数据平台独立融资空间被压缩"这个推论需要 5-6 月更多融资数据印证。
