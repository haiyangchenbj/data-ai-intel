趋势

月度趋势分析

趋势一：Agent 原生数据平台全面落地 ★★★（高可信）

论据：SAP（05-13 自治企业）+ 腾讯云（05-19 DataBuddy）+ Starburst（05-28 AIDA GA）+ dbt（05-19 Developer Agent）+ Databricks（05-27 Genie Code Agent 默认启用）+ Snowflake（05-27 Natoma 收购）+ Google Cloud（05-18 BigQuery Conversational Analytics GA）+ Microsoft（05-06 Fabric Data Agent）八家平台在同一月份以不同路径推进 Agent 原生架构，跨厂商、跨地域、跨定位，结构性证据充分。

5 月最大结构性变化是"Agent 可操作的数据平台"从概念验证走向产品发布。SAP 在 Sapphire 2026 将 Business Data Cloud、BTP 和 Business AI 合并为统一的 Business AI Platform，并与 AWS Athena 实现双向零拷贝数据共享，标志 ERP 巨头正式将数据平台定位为 Agent 底座。腾讯云发布 DataBuddy，用户通过自然语言完成数据接入、开发、治理、分析全链路，1-2 周数仓建设周期压缩至小时级，是国内首个 Agent 原生数据工作台。Starburst 从联邦查询引擎升级为 Enterprise Intelligence Platform，AIDA 通过 MCP 连接第三方工具触发工作流，不只是 text-to-SQL chatbot 而是 Agentic 控制面。

dbt Developer Agent 进入 Preview，用自然语言构建模型、生成 SQL/测试/文档，标志数据工程工具从"人写代码"进入"人提需求、Agent 交付"模式。Databricks Genie Code Agent 模式对合规工作区默认启用，单提示即可自动检索资产、生成运行代码、自动修错并可视化。Snowflake 收购 Natoma 填补 Agentic Enterprise 最后的治理短板——MCP 网关层的身份与策略控制。

Google Cloud 在 Next '26 发布 BigQuery Conversational Analytics GA（05-18），Gemini 驱动的 Agent 将自然语言问题转化为精确 SQL 并自动可视化，非技术用户可直接"对话"数据仓库。同期 Continuous queries GA 让 BigQuery 从批处理快照进化为实时流处理引擎，LookML Agent 则确保 Agent 输出与企业现有指标定义一致。Microsoft 在 Build 2026（05-06）以约 60 项公告完成数据平台全栈 Agent 化：Fabric Data Agent 让非技术用户用自然语言问答组织数据；Fabric Copilot 可生成查询并检测大规模数据中的未知异常；Real-Time Intelligence 创建统一的事件摄取-处理-路由场所。

So what：八家平台的路径差异揭示了一个共识：数据平台的下一个竞争维度不是"谁存的数据更多"，而是"谁能让 Agent 更可靠地操作数据"。数据工程、数据分析、数据治理三条传统管线正在 converged 为 Agent 可理解的统一语义层。Google 和 Microsoft 的加入意味着"Agent 原生数据平台"已从创业公司和新锐厂商的差异化策略，升级为全球三大云厂商（AWS/Azure/GCP）+ 两大独立平台（Databricks/Snowflake）+ ERP 巨头（SAP）+ 国内头部云（阿里云/腾讯云）的全行业共识。

趋势二：MCP 协议成为数据基础设施的标准控制面 ★★★（高可信）

论据：Confluent MCP Server GA（05-19）+ Informatica Headless IDMC via MCP（05-21）+ Snowflake 收购 Natoma（MCP 治理平台，05-27）+ OpenRouter 融资（Snowflake/Databricks/MongoDB Ventures 齐投，05-27/28）+ Starburst AIDA via MCP（05-28）+ Daloopa MCP 连接器（05-28），事件跨 6 家厂商、涵盖流处理/数据管理/推理路由/金融数据多个子领域。

MCP（Model Context Protocol）在 5 月从"一个有前景的开放协议"升级为数据基础设施的事实标准。Confluent 将 Managed MCP Server 推向 GA，AI 助手可通过自然语言管理 Kafka/Flink/Schema Registry，演示中全程自然语言完成 MySQL→Iceberg CDC 管道构建。Informatica 宣布整个 IDMC 以无头模式通过 MCP 同时接入 Databricks、Snowflake、Microsoft、AWS 四大平台，数据管理首次成为 Agent 可直接调用的标准化服务。

Snowflake 收购企业级 MCP 平台 Natoma，将身份治理与特权访问能力整合进 AI Data Cloud，让 Cortex Agents 通过统一 MCP 网关安全连接 Slack、Gmail、CRM 等企业系统。OpenRouter 完成 $113M B 轮融资，Snowflake Ventures、Databricks Ventures、MongoDB Ventures 三家数据平台厂商同时以战略投资方式跟投——这释放的信号已超越财务投资逻辑，推理路由/编排层被视为连接"数据"与"AI 消费"的关键中间件。

So what：对数据平台厂商而言，MCP 支持正从差异化卖点变为准入门槛。如果平台不支持 MCP，用户的 Agent 就无法管理你的产品。Snowflake 通过收购 Natoma 争夺 MCP 治理层的控制权，这一策略值得所有平台厂商关注。

趋势三：流处理与声明式数据工程成为 Agent 时代基础设施 ★★★（高可信）

论据：Confluent Real-Time Context Engine GA + Streaming Agents GA + dbt Adapter for Flink GA + Queues for Kafka GA（05-19/25）+ Databricks Runtime 18 Beta（Spark Declarative Pipelines + Real-Time Mode，05-25）+ Databricks Lakeflow Sinks GA（05-18），事件跨 Confluent/Databricks 两家核心厂商且互为补充。

Confluent 在 Current London 发布的"AI 数据流全家桶"标志着流处理平台的战略重定位：Real-Time Context Engine 将 Kafka topic 实时转化为 Agent 可通过 MCP 查询的结构化上下文层；Streaming Agents 是生产级事件驱动 Agent 直接运行在 Flink+Kafka 上，含 Agent 反思模式，99.99% SLA；Queues for Kafka（KIP-932）是 Kafka 15 年来最大的消费模型变化——原生支持队列语义，统一了 streaming 和 queuing 基础设施。Confluent AI 负责人 Sean Falconer 引用 McKinsey 报告："八成企业因数据层问题无法规模化 Agentic AI"。

Databricks Runtime 18 Beta 首次搭载 Apache Spark 4.1.0，Spark Declarative Pipelines（SDP）取代原 Delta Live Tables——用户只定义"要什么数据"而非"怎么跑"，引擎自动处理编排、故障恢复和增量处理，流批在同一声明式 API 下统一。Real-Time Mode 让 SDP 原生支持低延迟流处理，无需独立配置 Structured Streaming。同期 Lakeflow Sinks GA 支持直接输出到 Kafka/Event Hubs/自定义目标，消除 ETL 管道与外部系统之间的"最后一英里"断裂。

So what：Agent 系统需要的数据不是"昨天跑完的批处理结果"，而是"正在发生的实时上下文"。流处理平台从 ETL 替代方案升级为 Agent 的神经网络，声明式数据工程降低了 Agent"代写"数据管道的门槛。这个趋势将在 6 月双峰会得到进一步强化。

趋势四：国内数据基础设施政策与产品双轮驱动 ★★★（高可信）

论据：国家数据局《2026年数字经济发展工作要点》（05-20）+ 工信部工业数据 23 场景指引（05-07）+ 数字中国峰会 531 工具箱（05-07）+ 信通院《人工智能模数共振体系研究报告》（05-19）+ 阿里云全栈 Agent 化（05-20）+ 腾讯云 TBDS 重构 + DataBuddy（05-19/22）+ 华为 AI DC 方案（05-21），政策端与产品端同步推进、相互印证。

5 月国内 Data+AI 领域呈现罕见的政策与产品共振。国家数据局印发《2026年数字经济发展工作要点》，首次将"强化数据赋能人工智能发展"列为八大任务中的独立板块，提出六大专项行动（强基扩容、应用赋能、提质增效、管理服务、价值释放、标注攻坚），目标形成满足"AI 就绪度"要求的标杆性高质量数据集。央视报道披露两批数据基础设施先行先试已覆盖 15 个行业 / 43 个城市 / 1.9 万个生态主体。工信部同期发布《工业场景数据要素应用参考指引》，围绕 23 个典型场景凝练数据"采、集、用"路径。中国信通院在第九届数字中国建设峰会上发布《人工智能模数共振体系研究报告(2026年)》，首次系统定义"数据-模型双向驱动、协同优化"的技术体系，提出数据赋能模型迭代、模型反哺数据治理的闭环机制，为行业提供从"AI Ready"到"AI Native"的演进路径参考。

产品端，阿里云峰会宣布完成"芯片—Agentic Cloud—模型—推理"全栈 Agent 化升级，发布千问云、真武 M890 AI 芯片和 Qwen3.7-Max 模型，AI 收入占外部收入已首破 30%。腾讯云发布 DataBuddy 智能体工作台，TBDS 完成三层融合架构重构并与信通院 TC601 联合发布 Data+AI 白皮书。华为在巴黎 IDI Forum 发布 AI DC 数据基础设施全栈方案，CMS（Context Memory Storage）是业界首个支持异构算力的上下文记忆存储，可将推理首 Token 时延降低 90%。

So what："AI 就绪度"作为政策导向的质量标准，将倒逼数据平台产品在数据质量评估、标注管理、数据集版本控制等能力上升级。信通院"模数共振"框架的提出，意味着国内 Data+AI 发展已从单点技术突破进入"数据-模型协同"的系统化阶段。对面向政企客户的数据平台厂商，合规能力、政务数据对接能力以及与信通院标准体系的对接能力将成为新的差异化因素。

趋势五：开放数据格式进入可插拔多格式时代 ★★（中可信）

论据：Apache Iceberg 1.11.0 File Format API（05-22）+ Snowflake Iceberg v3 GA（05-11）+ Google Cloud BigQuery Iceberg 互操作预览（05-06）+ ClickHouse Rust Delta Kernel 集成（05-25），以 Iceberg 1.11.0 为核心事件、多条独立支持形成证据链。

Apache Iceberg 1.11.0 于 5 月正式发布，这不是常规维护版本而是结构性升级：File Format API 将 Iceberg 核心与物理存储格式解耦为插件模型，Vortex、Lance、Nimble 等下一代 AI 原生文件格式已在集成中；V3 Spec 特性生产就绪——Deletion Vectors 解决 V2 删除文件膨胀问题、Variant Type 支持半结构化数据、纳秒精度时间戳。这意味着单一表可以根据工作负载选择最优物理格式，而元数据层保持统一。

同期 ClickHouse 26.5 通过 Rust Delta Kernel 原生读取 Delta Lake 表，与既有 Iceberg 支持形成数据湖双格式覆盖；Google Cloud 预览 BigQuery Iceberg 读写互操作，Trino/Spark 等引擎可直接访问 BigQuery 托管表；Snowflake Iceberg v3 GA 新增删除向量和 Variant 类型。

So what：开放格式的竞争已从"是否支持 Iceberg/Delta"升级为"能否让 AI 原生格式与传统格式共存"。对于数据平台厂商，格式互操作性正在从差异化卖点变为准入门槛。Iceberg V4 规范的两项核心提案（相对路径、内容统计）已于 5 月进入社区投票，需要 6-7 月更多进展来验证这一趋势的加速度。
