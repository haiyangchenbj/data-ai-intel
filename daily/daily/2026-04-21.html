Top Signals

1. Databricks Data Classification 正式 GA：AI 自动分类敏感数据进入生产

来源：Databricks on AWS Release Notes / Azure Databricks Release Notes / Databricks on GCP Release Notes

Databricks 于 4 月 20 日将 Data Classification 功能从 Public Preview 推向 GA，跨 AWS、Azure、GCP 三个云平台同步发布。该功能使用 Agentic AI 系统自动扫描 Unity Catalog 中的表，识别并标记敏感数据（覆盖 PII、GDPR、HIPAA、DPDPA 等合规框架）。系统支持智能增量扫描（新表/新列 24 小时内自动分类）、自动标记（元存储级或目录级配置）、以及基于分类结果的 ABAC 策略自动生成。

为什么对数据平台重要：敏感数据分类此前是数据治理中最耗人力的环节。Databricks 用 Agentic AI + Unity Catalog 元数据的组合，将分类从"人工审查"变为"AI 自动执行 + 人工审核例外"，实质性降低了合规治理落地门槛。从 Public Preview 到 GA 意味着该方案已在生产环境验证过可靠性。这是 Databricks 治理层"Governed Tags（04-02 GA）→ Data Classification（04-20 GA）"连续落地的第二步，敏感数据治理正在从手动标签走向 AI 自动闭环。

2. Snowflake Cortex AI Guardrails 正式 GA：AI 内容安全护栏进入生产

来源：Snowflake Release Notes - New Features

Snowflake 于 4 月 20 日将 Cortex AI Guardrails 从 Preview 推向 GA。该功能为 Snowflake Cortex AI 平台提供内容安全防护层，基于 Meta Llama Guard 系列模型，通过 SNOWFLAKE.CORTEX.GUARD() SQL 函数对 LLM 输入和输出进行安全检测与分类，识别并过滤有害内容（暴力、仇恨言论、自残、不当内容等类别），返回安全/不安全的判断结果。

为什么对数据平台重要：随着 Snowflake 在 Cortex AI 上密集投入，平台上运行的 AI 工作负载正在快速增长。Guardrails GA 意味着企业现在可以在生产环境中对 Cortex AI 的每次调用都内建安全检查——这对金融、医疗、政府等受监管行业的采用至关重要。Guardrails 作为 SQL 原生函数嵌入平台，使 AI 安全不再是应用层的额外工作，而是数据平台基础设施的一部分。结合今天 Databricks 的 Data Classification GA，两大平台正在同步构建"可信 AI"的基础设施层。
