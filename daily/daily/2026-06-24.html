AWS 中国峰会 2026 开幕：数据库服务 VP 定位数据平台为 Agentic AI "上下文基础设施"

来源： 腾讯新闻
摘要： 2026 亚马逊云科技中国峰会 6 月 23 日在上海世博中心开幕，主题"Agentic Now"。亚马逊云科技数据库服务副总裁 Ganapathy "G2" Krishnamoorthy 在主旨演讲中提出 Agentic AI 的核心命题不再是模型能力，而是数据的上下文贯通——"上下文就藏在您的数据里"。G2 阐述了 AWS 四大 Agentic 方向：改变工作方式（跨系统上下文贯通）、守护安全（速度与安全不再二选一）、构建软件（从创意到生产代码以小时计）、打造专属 Agent（真正理解企业数据）。同期发布了 Amazon Quick 知识图谱能力、Amazon Continuum 自动化安全运营服务、Agent Core 网络搜索等。
为什么对数据平台重要： AWS 将数据库服务 VP 而非 AI/ML VP 推上中国峰会主舞台，本身就是数据平台在 Agentic AI 时代战略地位的明确信号。G2 的核心论点——"Agent 的第十个决策应比第一个更好，而上下文来自数据"——重新定义了数据平台不是 Agent 的后端存储，而是 Agent 智能复利的引擎。这一叙事与 Databricks（Genie Ontology 上下文层）和 Snowflake（Horizon Context）在各自峰会上的方向完全一致：三大平台正在围绕"数据即 Agent 上下文基础设施"展开新一轮角逐。

dbt 平台多项行为变更标志正式生效，Analyst 只读权限集进入私有 Beta

来源： dbt 官方文档
摘要： dbt 平台"Latest"发布轨道的多项行为变更标志于 6 月 23 日正式成熟（默认启用），包括 `skip_nodes_if_on_run_start_fails`、`state_modified_compare_more_unrendered_values`、`require_yaml_configuration_for_mf_time_spines` 等，影响 dbt 项目的运行行为、状态比对和语义层配置。同期新增 Analyst 只读权限集（私有 Beta，支持通过 OAuth 连接 MCP Server 等分析功能）、Workspace 级 Microsoft Fabric Private Link（Beta）、Cost Insights 扩展至 Amazon Redshift。
对数据平台的影响： 行为变更标志集体成熟标志着 dbt 平台从"建议最佳实践"进入"强制执行标准化"阶段——历史遗留配置将不再兼容默认行为，企业级 dbt 项目需在成熟前完成适配。Analyst 只读权限集的引入使 dbt 从数据工程师工具向全组织数据平台演进，非技术用户可通过 OAuth + MCP Server 直接消费 dbt 项目元数据和分析结果。
