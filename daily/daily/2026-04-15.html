1. Databricks 开放 Unity Catalog Delta 表的外部写入能力，外部 Spark 可直接创建和写入托管表（Beta）

来源：Azure Databricks Release Notes - April 2026

Databricks 于 4 月 14 日发布 Unity REST API 的重要扩展：外部 Delta 客户端（如 Apache Spark）现可对 Unity Catalog 的外部表（external）进行完整读写创建操作，托管表（managed）的创建和写入功能进入 Beta 阶段，需启用 catalog-managed commits。这意味着运行在 Databricks 之外的 Spark 集群可以直接向 Unity Catalog 注册并写入 Delta 表。

为什么对数据平台重要：这是 Unity Catalog 从"Databricks 内部治理层"向"跨引擎数据控制平面"演进的关键一步。此前外部引擎只能通过 Iceberg REST Catalog 以只读方式访问 Unity Catalog 数据；现在写入通道也打开了，Unity Catalog 有可能成为多引擎环境下的统一数据注册中心。对于运行混合 Spark 环境（EMR/Dataproc + Databricks）的企业而言，这降低了数据孤岛形成的风险。

2. Snowflake 推出 Snowflake Storage for Apache Iceberg Tables（Preview），用自有存储承接 Iceberg 表

来源：Snowflake Release Notes - April 2026

Snowflake 于 4 月 14 日宣布 Snowflake storage for Apache Iceberg tables 进入 Preview 阶段。此前 Snowflake 的 Iceberg 表仅支持用户自管理的外部存储（S3/GCS/Azure Blob）；现在 Snowflake 可以使用自有管理的存储来承载 Iceberg 表数据，用户无需配置和维护外部存储卷。这使 Iceberg 表在 Snowflake 内的使用体验接近原生表。

为什么对数据平台重要：这代表了 Snowflake 对 Iceberg 生态的策略升级——从"兼容开放格式"到"用平台存储收编开放格式"。用户获得了更简单的运维体验，但代价是数据存储重新回到 Snowflake 控制范围内。与 Databricks 同日开放外部写入的方向正好相反：Databricks 让数据可以从外部写入平台，Snowflake 让开放格式的数据回到平台内部。两者的路径分化正在定义开放表格式竞争的下一阶段。

1. 【跟踪】Rockstar Games 确认数据泄露：ShinyHunters 通过第三方 SaaS 平台 Anodot 入侵 Snowflake 实例

来源：Bitdefender - Rockstar Games confirms data breach

为什么值得继续看：ShinyHunters 声称通过 AI 驱动的 SaaS 平台 Anodot 的漏洞获取了 Rockstar Games 的 Snowflake 实例访问权限，窃取近 8000 万条记录。4 月 14 日为勒索截止日期，Rockstar 已确认泄露发生。这一事件延续了 2024 年 Snowflake 客户连环泄露事件的模式——攻击路径不经过 Snowflake 本身，而是通过集成的第三方工具。对数据平台安全架构的启示是：平台自身的安全边界不等于数据安全边界，SaaS 供应链正在成为数据平台最薄弱的环节。

需要等待什么信号确认：Snowflake 是否发布针对第三方集成安全的产品更新或最佳实践指引；此次事件的泄露范围和影响是否推动数据平台厂商收紧第三方 OAuth 凭证管理机制。

2. 【跟踪】Databricks 与 Snowflake 的开放表格式互操作路径分化：开放写入 vs 平台内聚

来源：Azure Databricks Release Notes、Snowflake Release Notes

为什么值得继续看：4 月 14 日同一天，Databricks 开放 Unity Catalog Delta 表的外部写入（让外部引擎成为一等写入者），Snowflake 推出自有存储承载 Iceberg 表（让开放格式回到平台管控下）。这两个方向的分化不是偶然——Databricks 的策略是做多引擎生态的中央控制平面，Snowflake 的策略是用极简体验将开放格式用户留在平台内。这两条路线在 Iceberg V3 时代会如何演化，直接影响企业数据架构的供应商锁定程度和迁移成本。

需要等待什么信号确认：Databricks 外部 Delta 写入从 Beta 到 GA 的时间表；Snowflake Iceberg 自有存储 Preview 后的 GA 路径及是否支持外部引擎直接访问这些表。
