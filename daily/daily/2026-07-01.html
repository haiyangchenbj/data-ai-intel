Product & Tech

Snowflake Dynamic Iceberg Table Replication 正式 GA

来源： Snowflake Release Notes
摘要： Snowflake 宣布动态 Iceberg 表的复制能力正式 GA。该功能支持将动态 Iceberg 表纳入复制和故障转移组，此前动态 Iceberg 表在复制刷新操作中会被跳过。此次 GA 意味着企业可以在跨区域灾备架构中完整覆盖 Iceberg 原生表。
对数据平台的影响： Iceberg 表格式的复制能力 GA 是开放格式在企业级灾备场景走向成熟的关键一步。对于已采用 Snowflake Iceberg 表的企业，不再需要在灾备方案中为 Iceberg 表单独设计复制逻辑，降低了开放格式在多区域部署中的运维复杂度。这也进一步缩小了 Iceberg 原生表与 Snowflake 专有表在企业级能力上的差距。

Confluent Cloud 托管连接器新增自定义 SMT 与 mTLS 支持

来源： Confluent Cloud Release Notes
摘要： Confluent Cloud 于 6 月 30 日发布两项连接器能力升级：全托管连接器（Basic/Standard/Dedicated 集群）现已支持自定义单消息转换（Custom SMT），允许用户在托管连接器中注入自有转换逻辑；同时 PostgreSQL 和 MySQL 的 JDBC 连接器新增 mTLS 双向认证支持。
对数据平台的影响： Custom SMT 的开放意味着 Confluent Cloud 的托管连接器不再局限于内置转换能力，企业可以将此前需自建 Kafka Connect 集群才能运行的定制逻辑迁移到全托管环境。mTLS 的支持则消除了金融、医疗等强合规行业在采用托管连接器时的认证障碍。两项更新共同指向一个趋势：托管流平台正在系统性吸收此前属于自建基础设施层的能力。

【收购兼并】Parabellum Investments 收购数据基础设施公司 Crux Informatics

来源： PR Newswire
核心数据： 交易金额未披露。Crux 服务客户包括 Goldman Sachs、Two Sigma、Google Cloud，管理数千条生产客户数据管道，集成 200+ 数据供应商（Morningstar、Moody's、MSCI 等）。
摘要： PE 机构 Parabellum Investments 于 6 月 30 日宣布收购 Crux Informatics。Crux 是金融行业外部数据管理的核心基础设施，通过 Sphere（全周期托管数据服务）和 ArrayX（Agentic 数据管道平台）两条产品线，为金融机构提供从原始数据到可用洞察的自动化管道。该平台深度集成 Databricks、Snowflake 和 Google Cloud。
对数据平台的影响： Crux 被 PE 收购是数据管道中间件进入平台化整合期的信号。随着 Agent 驱动的金融分析场景增长，外部数据的自动化接入、清洗和治理正从自建能力变为采购决策中的独立品类。PE 的进入意味着这个中间件层将加速产品化和独立运营，对 Databricks/Snowflake 等平台生态中的"数据接入最后一公里"供应商格局产生影响。
