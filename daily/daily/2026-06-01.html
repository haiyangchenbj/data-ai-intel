1. Snowflake Q1 FY27 产品收入同比增长 34%，签约 AWS $60 亿协议并收购 MCP 平台 Natoma

来源：Snowflake Investor Relations

Snowflake 2027 财年 Q1（截至 2026-04-30）总营收 $13.9 亿（+33%），产品收入 $13.3 亿（+34%），创历史最强环比美元增量。RPO $92.1 亿（+38%），NRR 126%，百万美元客户 779 家（+29%）。全年产品收入指引从 $56.6 亿上调至 $58.4 亿。与 AWS 签署五年 $60 亿战略协议（含 Graviton 与 AI GPU 集成），签署收购企业 MCP 平台 Natoma 最终协议。盘后股价飙涨约 37%。

💡 产品收入加速且 RPO 增速高于收入增速，证明 AI workload 正在转化为有粘性的数据平台消费承诺。$60 亿 AWS 协议锁定算力-存储深度绑定路径；Natoma 收购标志 MCP 协议从开发者工具向企业 Agent 治理层渗透。

2. Anthropic 完成 $650 亿 H 轮融资，投后估值 $9650 亿超越 OpenAI

来源：Anthropic 官方新闻稿

Anthropic 5 月 28 日宣布完成 $650 亿 H 轮融资，由 Altimeter Capital、Dragoneer、Greenoaks 和红杉资本领投，投后估值 $9650 亿。年化收入突破 $470 亿。同时签署与 Amazon 5GW 新容量、Google+Broadcom 5GW TPU 容量协议。存储三巨头美光、三星、SK 海力士作为战略伙伴参投。

💡 $470 亿 ARR 意味着 Claude 正嵌入企业核心工作流，10GW 算力协议推动对 HBM、高性能存储和数据管道的天量需求。存储巨头入股标志 AI 模型层与数据基础设施供应链深度耦合。

3. Google Cloud 预览 BigQuery 跨引擎 Apache Iceberg 支持，统一治理跨云 Lakehouse

来源：Google Cloud Blog / InfoQ 报道

Google Cloud 5 月 31 日正式预览 BigQuery 跨引擎 Iceberg 支持：同一 Iceberg 表可由 BigQuery、Spark、Flink、Trino 等多引擎直接读写无需复制数据。核心能力包括托管元数据与自动表维护、跨 AWS/Azure/Databricks/Snowflake 的 Catalog 查询、统一 IAM 权限管控、ObjectRefs 多模态数据引用。

💡 Google 将 BigQuery 从"分析引擎"重新定位为"跨云 Lakehouse 治理中台"。Catalog 和元数据管理成为新锁定层——"谁管 Catalog 谁掌握入口"的竞争格局正在 Google/Databricks Unity/Snowflake Polaris 三方间形成。

Product & Tech

1. 腾讯云数据库面向 Agent 全面升级：发布 Agent Memory、DatabaseClaw，TDSQL-B 架构级重构

来源：环球网 / 腾讯云官方

5 月 29 日，腾讯云宣布数据库产品体系面向 Agent 场景全面升级。核心发布：(1) Agent Memory——三层记忆服务（短期压缩/长期沉淀/团队记忆），任务成功率+30%，最高节省 60% Token 成本，开源一周获 4000+ Stars；(2) DatabaseClaw——基于 OpenClaw 的数据库 AI Agent，覆盖 14 款产品 1600 API；(3) TDSQL-B 架构级重构，统一事务/向量/全文/图计算四引擎；(4) TDSQL-C 引入数据库 Branch 能力，1TB 秒级分叉。

💡 腾讯云将数据库从"被动存储"重新定位为"Agent 原生数据底座"。Agent Memory "团队记忆"概念在业内属首批产品化落地；TDSQL-B 四引擎统一对标多模数据库趋势，Database Branch 对标 Neon/PlanetScale。

2. Databricks Lakeflow Real-Time Mode 进入 Public Preview：端到端延迟低至 5ms

来源：Databricks Release Notes

5 月 27 日，Databricks 宣布 Lakeflow Spark Declarative Pipelines 的 Real-Time Mode 进入 Public Preview。该模式可实现端到端延迟低至 5 毫秒，适用于欺诈检测、实时个性化、即时决策等场景。使用新增的 update_flow 装饰器，支持 update output mode 和无需 watermark 的有状态聚合。

💡 Databricks 在流批一体架构中补齐"极低延迟"短板。5ms 延迟直接对标 Flink 核心优势场景，Spark 生态用户无需独立流引擎即可覆盖实时决策类 workload。
