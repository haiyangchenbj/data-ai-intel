1. Snowflake 发布开放互操作框架：Iceberg v3 GA + Polaris 双向 Catalog + Horizon 统一治理

来源：Snowflake 官方新闻稿

Snowflake 在 Summit 26 Day 2 宣布以 Apache Iceberg v3 为核心构建完整的开放互操作栈。关键组件包括：Iceberg v3 正式可用（GA），支持更多数据类型、跨系统变更追踪与半结构化数据高性能处理；Snowflake Storage for Apache Iceberg Tables GA，为开放数据提供完全托管体验；Horizon Catalog（由 Apache Polaris 驱动）GA，实现跨 Snowflake、外部数据湖和开放系统的双向读写与统一治理。此外，外部引擎访问管理、Iceberg REST Scan Plan API 等进入预览，确保细粒度保护跨兼容引擎生效。

为什么对数据平台重要：这是主流云数据平台首次以完整产品栈级别拥抱开放表格式互操作，从存储层到目录层再到治理层全部打通。意味着企业可以在 Snowflake 内部和外部引擎之间共享一份实时、受治理的数据副本，无需移动或复制数据。对架构选型而言，"开放格式避免锁定"从口号变成了可落地的工程现实，将直接影响企业的多云策略和采购决策。

2. Immuta 联合 Snowflake 推出 Agentic Data Access：AI Agent 会话级数据访问治理

来源：PR Newswire

数据访问治理厂商 Immuta 在 Snowflake Summit 26 上发布三项由 Snowflake Cortex AI 驱动的新能力。Immuta Agentic Data Access 使 AI 代理通过自然语言与企业数据交互，Cortex 负责查询规划，Immuta 在会话级别强制执行访问边界，为每个代理交互分发唯一的临时角色，范围限定于授权用户的权限。Agent Principal Context 扩展 Snowflake 的代理主体上下文，治理 AI 代理对 Snowflake 外部数据的出站访问，采用零常驻权限的动态配置。Comply App 为 Horizon Catalog 提供自然语言合规查询界面。

为什么对数据平台重要：这是数据治理领域首个面向 AI Agent 时代设计的访问控制方案。传统的基于角色的静态权限模型无法应对 Agent 自主决策、多步骤推理、跨系统调用的场景。Immuta 的"会话级临时角色 + 双重身份审计追踪"架构，将治理粒度从"人能访问什么"推进到"Agent 在代表谁、能访问什么、每一步是否合规"。对数据平台而言，Agent 治理将成为与 Agent 能力同等重要的基础设施。

3. Workday Data Cloud 与 AWS 深度集成：双向零拷贝 + Agent Gateway 跨云数据访问

来源：PR Newswire via Finviz

Workday 在 DevCon 大会宣布 Workday Data Cloud 将与 AWS 深度集成，开发者可从 AWS 工具和服务双向零拷贝访问 Workday 治理的 HR 与财务数据，无需构建自定义管道或复制数据。基于 Apache Iceberg 的 Workday Data Connect 提供预构建连接器；Workday Agent Gateway 使 AWS 上构建的 AI 代理能安全访问 Workday 的薪资、福利和财务数据，复用现有治理、权限和审计控制。Workday Data Lake、Data Connect 和 Live Data Query 已向早期采用者开放，AWS 集成功能将在未来几个季度通过早期访问计划逐步推出。

为什么对数据平台重要：这是企业 SaaS 数据与云数据平台深度集成的标志性案例。传统上，SaaS 数据通过 API 或 ETL 管道导出到数据仓库，存在延迟高、治理断裂、重复存储等问题。Workday 的"双向零拷贝 + 治理随行"模式意味着 SaaS 数据可以像云原生数据一样被直接查询和治理，AI 代理也能在保持合规的前提下实时访问业务上下文。对数据平台架构而言，SaaS 数据层与云数据平台的边界正在消融。

Product & Tech

1. Collibra 扩展与 Snowflake 集成：Horizon Catalog 与 Cortex AI 双向语义治理

来源：新浪财经

数据治理厂商 Collibra 在 Snowflake Summit 26 上宣布扩大与 Snowflake 的集成，在 Snowflake AI Data Cloud 中提供可信业务上下文和语义一致性。Collibra 的企业上下文流入 Snowflake，实现跨 Horizon Catalog 和 Cortex AI 的双向集成，涵盖治理与策略增强、AI 语义增强、自动可追溯性、可扩展数据访问管理等方面。Collibra AI Command Center 为 Snowflake 上的自主 AI 提供控制层。此次集成已向部分共同客户提供预览版，计划 2026 年第三季度全面推出。

对数据平台的影响：数据目录与数据治理平台的深度集成，解决了 AI 时代"业务语义一致性"的核心难题。当 AI 代理查询数据时，如果不能理解企业特定的指标定义（如"毛利率"在该公司如何计算），就会做出错误决策。Collibra 的上下文层与 Snowflake 的计算层结合，使治理语义可以随数据一起被 Agent 理解和使用，是数据平台从"数据可用"走向"数据可信"的关键一步。
