Top Signals

1. LangChain 发布 SmithDB：首个面向 Agent 可观测性的专用分布式数据库

来源：LangChain Blog / Interrupt 2026 大会

LangChain 在 Interrupt 2026 大会（5月13-14日，旧金山）上发布 SmithDB，这是一个用 Rust 构建、基于 Apache DataFusion 和 Vortex 的分布式数据库，专门为 Agent 可观测性场景设计。SmithDB 作为 LangSmith 平台的数据层，解决了 Agent 运行时产生的大规模追踪数据的存储、查询和分析问题。同日还发布了 LangSmith Engine 和 LangSmith Context Hub。

💡 Agent 可观测性正在从通用日志/APM 工具中独立出来，催生专用数据基础设施。SmithDB 选择 DataFusion + Vortex 的列式存储架构而非传统 OLAP，说明 Agent trace 数据的查询模式（高基数、深嵌套、跨 session）需要新的存储和计算引擎。这对数据平台厂商是一个信号：Agent 数据管理可能是下一个需要专门解决方案的领域。

2. Snowflake 推出敏感数据权限报告（Public Preview）：数据治理从分类走向权限可见性

来源：Snowflake Release Notes 2026-05-13

Snowflake 发布 Sensitive Data Entitlement Report（公开预览），允许管理员生成报告查看哪些用户角色拥有对敏感数据表的访问权限。该功能将数据分类结果（哪些列包含敏感数据）与访问控制（谁能看到这些数据）关联起来，生成一张权限-敏感数据交叉视图。

💡 数据治理的核心痛点正在从"发现敏感数据在哪里"演进到"谁能访问敏感数据"。这是 Snowflake 将 Horizon（治理框架）从被动分类推向主动权限洞察的关键一步。对于受监管行业（金融、医疗），这类功能将从"锦上添花"变为平台选型的硬性门槛。

Product & Tech

1. Snowflake Cortex AI 函数支持细粒度权限控制（Preview）

来源：Snowflake Release Notes 2026-05-13

Snowflake 新增 USE AI FUNCTION 细粒度权限，管理员可按函数级别控制角色对 Cortex AI Functions 的调用权限，替代此前"全有或全无"的 USE AI FUNCTIONS 权限。

💡 AI 函数的精细化权限管理是数据平台将 AI 能力纳入企业治理体系的关键一步。当 AI 函数成本和安全风险差异显著时（如 LLM 推理 vs 嵌入生成），按函数授权比按角色授权更贴近企业的实际管控需求。

2. Databricks ai_parse_document 在合规安全配置工作区默认可用

来源：Databricks Release Notes May 2026

Databricks 的 ai_parse_document 函数现已在启用合规安全配置（HIPAA、PCI-DSS、FedRAMP Moderate 等）的工作区默认可用。该函数可从 PDF、图片、Word、PowerPoint 等非结构化文档中解析结构化内容。

💡 非结构化文档解析进入合规场景（医疗、金融、政府），意味着 Databricks 正在将 AI 数据处理能力从实验性功能推向受监管行业的生产使用。这降低了合规行业采用 AI 增强数据管道的门槛。
