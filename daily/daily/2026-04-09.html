Product & Tech

1. Databricks Runtime 18.2（Beta）发布：Spark 4.1.0 驱动，Structured Streaming 支持倒带重放

来源：Databricks on AWS Release Notes - April 2026

Databricks 4 月 8 日发布 Runtime 18.2（Beta），由 Apache Spark 4.1.0 和 Delta Lake 4.1.0 驱动。关键更新包括：Structured Streaming 支持倒带和重放（从流的较早点重新处理数据以从故障中恢复）、Delta 表原生支持 NullType (VOID) 列、Unity Catalog 强制执行 SQL UDF 依赖验证以防绕过访问控制、AWS SDK v1 依赖 shading 为 v2 迁移做准备。ML 版同步发布，包含 PyTorch 2.9.0、CUDA 12.9、Transformers 4.57.1 等。

对数据平台的影响：Structured Streaming 倒带重放是实时数据管道工程的关键能力缺口补齐——此前从模式变更、数据格式错误中恢复通常需要完全重置状态，现在可以精确回溯至故障点重新处理。Unity Catalog UDF 依赖验证的强制执行表明 Databricks 正在系统性收紧治理层的安全边界。AWS SDK v1 shading 预示着未来版本将全面切换至 v2，使用旧 SDK 的用户应提前准备迁移。

2. Databricks 作用域个人访问令牌（Scoped PAT）正式 GA

来源：Databricks on AWS Release Notes - April 2026

Databricks 4 月 8 日宣布作用域个人访问令牌（Scoped PAT）正式 GA。用户现在可以将 PAT 限制为特定的 API 操作，而非此前的全权限令牌，实现最小权限原则（Principle of Least Privilege）。

对数据平台的影响：对于在 CI/CD 管道、自动化脚本和第三方集成中广泛使用 PAT 的数据团队，作用域限制意味着即使令牌泄露，攻击面也被限定在预设的 API 范围内。这是 Databricks 安全治理体系从"粗粒度"向"细粒度"演进的标志性一步，与企业安全合规要求（SOC 2、HIPAA 等）的对齐度显著提升。

3. Snowflake Snowpipe Streaming 错误日志正式 GA

来源：Snowflake Release Notes

Snowflake 4 月 8 日宣布 Snowpipe Streaming 错误日志功能正式 GA。该功能为 Snowpipe Streaming 引入结构化的错误日志记录能力，使数据工程团队能够追踪和诊断流式摄取过程中的错误。

对数据平台的影响：Snowpipe Streaming 是 Snowflake 实时数据摄取的核心通道，此前流式加载过程中的错误定位依赖间接手段。错误日志 GA 意味着 Snowflake 的流式摄取进入生产级可观测性阶段，对于构建关键业务实时管道的团队，故障排查和 SLA 保障能力获得实质性提升。
