1. ClickHouse Open House 2026：Agent、Postgres 与 Observability 三连发

来源：ClickHouse Cloud Changelog
ClickHouse 在 6 月 3 日的 Open House 2026 活动中密集发布四项重大更新。ClickHouse Agents 进入 Public Beta，基于 Claude 提供对话式 schema-aware 分析，支持无代码 AgentBuilder、MCP 服务器插件和子代理工作流。ClickHouse Postgres 同步进入 Public Beta，在本地 NVMe 存储上提供托管 Postgres 服务，声称 TPS 较 AWS RDS 提升 5 倍以上，并通过原生 CDC 管道将数据实时同步至 ClickHouse。ClickStack Cloud 作为 serverless 可观测性平台启动 Private Preview，基于 OpenTelemetry 提供日志、指标和追踪的统一管理。
为什么对数据平台重要：ClickHouse 首次将 AI Agent 能力作为第一方功能嵌入分析数据库，标志着查询引擎与 AI 推理层开始深度融合。Postgres + ClickHouse 的托管组合则打通了 OLTP 到 OLAP 的实时链路，无需中间件。这两项变化共同指向一个趋势：分析型数据库正在从"被动响应查询"转向"主动理解业务并执行动作"。

2. Microsoft Build 2026：Fabric Data Factory 引入 MDF Transform 与私有网络镜像

来源：Microsoft Fabric Updates Blog
Microsoft 在 Build 2026 大会上宣布 Fabric Data Factory 多项更新。Mapping Data Flows (MDF) Transform 将在 6 月 8 日当周进入 Dataflow Gen2，提供低代码 Spark 级数据工程体验。Secure ingestion into Fabric Data Warehouse with Copy job/activity 正式 GA，支持最严格网络拓扑下的工作空间私有链路。Mirroring with Workspace Private Link 进入 Preview，覆盖 Azure SQL DB、SQL Server、SAP Datasphere 和 SharePoint List。此外，Rayfin 开源 SDK 发布，支持将 Agent 构建的应用直接部署到 Fabric 平台作为受治理的生产后端。
为什么对数据平台重要：MDF Transform 进入 Dataflow Gen2 意味着微软将 Azure Data Factory 的核心能力下沉到 Fabric 的 SaaS 体验中，降低了大规模数据工程的入门门槛。Rayfin 的发布则表明微软正在构建 Agent 应用与数据平台之间的双向闭环：Agent 从 Fabric 取数据，应用产出的数据又回流到 Fabric。这一架构对数据平台的治理模型和实时性提出了新要求。

3. Snowflake Summit Day 3：CoWork 详细功能曝光，Cortex Training 首次亮相

来源：Snowflake 官方 / 163.com 报道
Snowflake 在 Summit Day 3 期间披露了 CoWork（原 Snowflake Intelligence）的完整功能细节。CoWork 新增 Artifacts（可发布仪表盘）、Cortex Sense（为 Agent 提供共享业务上下文层）、User Skills（自动化工作流）、User Memory（个性化学习）和 Skill Catalog。Cortex Training 首次亮相，提供托管 GPU 环境供企业在 Snowflake 内定制基础模型。Snowflake 同时透露已有超过 13,900 家客户，其中 13,600 多个账户每周使用其 AI 产品。
为什么对数据平台重要：CoWork 的完整功能揭示了 Snowflake 对"Agentic Enterprise"的具体定义：不是单一 Chatbot，而是由共享上下文层（Cortex Sense）连接的可组合 Agent 网络。Cortex Training 则意味着 Snowflake 从"数据平台"向"模型训练平台"延伸，客户无需将敏感数据移出即可微调模型。这一组合使 Snowflake 成为同时覆盖数据、治理、模型训练和 Agent 部署的完整栈。

Product & Tech

1. AWS SageMaker Unified Studio 新增 Notebook 调度和 Data Agent 对话历史

来源：AWS Release Notes
AWS 于 6 月 3 日为 SageMaker Unified Studio 发布两项更新。Notebook scheduling 支持直接从笔记本界面调度、参数化和编排笔记本运行，无需外部编排基础设施，并支持多笔记本工作流链式执行。SageMaker Data Agent 新增 90 天对话历史保留功能，支持恢复之前的分析会话。
对数据平台的影响：Notebook scheduling 的引入降低了数据科学工作流自动化的门槛，使分析师可以直接将探索性分析转化为生产级定时任务。Data Agent 的对话历史则增强了可观测性和审计能力，对于受监管行业的 AI 辅助分析尤为重要。两者共同强化了 AWS 在"数据工程 + AI"一体化方面的竞争力。

2. NetApp 与 Cisco 扩展合作，将存储层纳入 SOAR 自动化响应

来源：Business Wire
NetApp 与 Cisco（通过 Splunk）于 6 月 3 日宣布扩展合作，发布 NetApp Splunk SOAR Playbook。该方案允许 Splunk SOAR 在检测到勒索软件威胁时，直接在 NetApp ONTAP 存储层执行自动化响应动作，包括阻断可疑用户、创建快照和下线数据卷，无需人工干预。
对数据平台的影响：这是数据基础设施与安全运营深度集成的典型案例。传统上，存储层是安全响应的盲区——检测到威胁后需要人工介入存储系统。将 SOAR 能力延伸到存储层意味着数据平台本身成为主动防御体系的一部分，这对金融、医疗等强合规行业的数据架构设计具有参考意义。

1. Gartner：2026 数据与分析十大预测

来源：Gartner 中国新闻室
核心观点：Gartner 发布 2026 年及未来数据与分析领域十大预测。关键判断包括：到 2027 年末，GenAI 与 AI 智能体的应用将给主流生产力工具带来 30 年来首次真正挑战，引发价值 580 亿美元的市场格局重塑；到 2029 年，AI 智能体从物理环境产生的数据量预计将达到所有数字 AI 应用数据总和的 10 倍；到 2030 年，50% 的企业将使用自主 AI 智能体将治理政策转化为机器可验证的数据合约。
映射到数据平台的判断：Gartner 的预测明确了数据平台未来的两个核心命题：一是 Agent 产生的数据规模将远超人类产生的数据，平台必须为此设计新的存储、治理和实时处理架构；二是治理将从"人工制定规则"转向"机器自动执行合约"，这意味着数据平台需要内建策略引擎和自动化合规能力。对于当前正在规划 Agent-Ready 架构的企业，这些预测提供了 3-5 年的技术路线图参考。

Capital & Corporate

投融资Coralogix 完成 2 亿美元 F 轮融资，总融资达 5.5 亿美元

来源：Coralogix 官方博客
核心数据：Series F 融资 2 亿美元，由 Advent 和 CPPIB 领投，Greenfield 和 Brighton Park Capital 跟投。总融资额达 5.5 亿美元。

Coralogix 于 6 月 3 日宣布完成 2 亿美元 F 轮融资。公司定位为"AI 时代的可观测性数据平台"，其流式引擎和 schema-free 架构支持全保真摄取和实时分析，内置 AI Agent Olly 可自动执行事件调查和根因分析。CEO Ariel Assaraf 表示，Coralogix 的架构在 Agent 时代"不需要重新定位，因为它就是为这个场景构建的"。
对数据平台的影响：Coralogix 的高估值反映了市场对"AI-Native 可观测性"的强烈需求。传统可观测性工具基于采样和仪表盘设计，难以应对 Agent 产生的大规模遥测数据。Coralogix 的全保真流式架构为数据平台领域提供了一个重要参照：当 AI 系统成为数据的主要消费者时，存储和查询架构需要从"人类可读"转向"机器优先"。
