OceanBase发布湖库一体AI数据库，国产数据库重新定位为AI时代的Agent数据基础设施

来源：PRNewswire
摘要： 6月29日，OceanBase发布面向AI时代的湖库一体AI数据库，提出LakeBase架构，将数据湖的开放存储能力、数据库的事务处理与分析能力、多模态数据处理能力统一到一套强一致的数据底座上，帮助Agent一次性获取完整业务上下文。同步发布三层产品体系：Lakebase（底层引擎统一管理结构化/非结构化/向量数据）、DataStudio（数据治理与服务化平台）、DataPilot（自然语言驱动的业务智能Agent）。该方案较传统多系统架构可降低整体TCO约30%-50%，已在蚂蚁阿福、灵光等AI场景完成验证，其中灵光累计生成数千万个"闪应用"，验证了千万级Agent场景的可行性。
为什么对数据平台重要： OceanBase此举标志着中国数据库厂商在AI时代数据基础设施方向上的战略级转向——不再只是"兼容和追赶"，而是主动定义"AI需要什么样的数据库"。LakeBase架构直接回应了当前企业AI落地的核心瓶颈（Agent需要统一上下文而非碎片化数据源），将数据库从"系统记录"重新定位为"AI可信上下文引擎"。CEO杨冰明确表示"从跟随者走向共同定义者"，叠加OceanBase连续两年中国分布式数据库本地部署市场第一和同时登顶TPC-C/TPC-H的技术信用，这一发布对国内数据平台采购选型和架构决策有实质性影响。

Databricks将LLM治理纳入Unity Catalog，数据目录从管数据延伸到管模型

来源：Databricks Release Notes (June 2026)
摘要： 6月29日，Databricks通过Unity Catalog的Model Services功能（Beta）将LLM端点作为可治理资产纳入数据目录。Model Service是Unity Catalog中的可安全对象，代表受治理的LLM端点，允许组织定义一次端点并通过Unity Catalog权限跨workspace共享，替代每个workspace重复配置端点。Databricks在system.ai schema中提供预置的Model Services用于托管基础模型，用户也可通过Unity AI Gateway UI、Catalog Explorer或REST API自行创建。
为什么对数据平台重要： 这是数据治理平台向AI治理层延伸的标志性信号。Unity Catalog过去管的是表、视图、模型文件等传统数据资产，现在开始管LLM端点这一AI原生资产。这意味着企业数据平台的治理边界正在系统性地向AI推理层扩展，"数据治理"和"AI治理"的界限在平台层面开始消融。对于正在构建Agent系统但苦于AI治理碎片化的企业来说，在数据平台内统一治理数据+模型的路径开始具备产品化可行性。

Databricks SQL alert task在Lakeflow Jobs中正式GA

来源：Databricks Release Notes (June 2026)
摘要： 6月29日，Databricks SQL alert task在Lakeflow Jobs中正式GA。该任务类型可在Lakeflow Job中评估Databricks SQL alert，并将评估状态作为任务输出值返回，下游任务可根据结果进行分支处理。
对数据平台的影响： 将SQL告警纳入数据工程编排工作流是Lakeflow Jobs功能完整性的重要补全，使数据团队能基于数据质量/业务指标监控结果自动触发下游处理（如数据修复、通知推送、管道暂停），减少了围绕数据可观测性搭建外挂编排的工程负担。
