Product & Tech

1. Databricks 修改 Catalog/Schema 托管存储位置功能正式发布（GA）

来源：Databricks on AWS Release Notes / Azure Databricks Release Notes — April 17, 2026

Databricks 于 4 月 17 日将修改 Unity Catalog 中 catalog 或 schema 托管存储位置的功能正式 GA。托管存储位置决定了新建托管表和卷的默认云存储路径，此前该功能仅支持创建时指定、不可变更。

对数据平台的影响：存储位置的可变更性在多云、多区域部署场景下直接影响数据主权合规和成本优化。此前如果需要变更存储路径，只能重建 catalog/schema 并迁移数据。GA 后，平台管理员可以在不中断服务的情况下调整存储布局——对正在进行云迁移或多区域扩展的企业而言是实质性的运维改善。

2. Microsoft 从 Power BI Databricks 连接器中移除 BI 兼容模式

来源：Azure Databricks Release Notes — April 17, 2026

Microsoft 于 4 月 17 日从 Power BI 的 Azure Databricks 连接器中移除了 BI 兼容模式选项。使用该选项的报告将不再正常工作。Databricks metric views 在 Power BI 中的支持受到直接影响，用户被引导至 Fabric Ideas 论坛提交反馈。BI 兼容模式在其他 BI 工具中仍然可用。

对数据平台的影响：这是一个 breaking change——已部署的 Power BI 报告如果依赖 BI 兼容模式将直接失效。表面上是连接器的技术调整，但背后反映的是 Microsoft 推动 Power BI 用户向 Fabric 原生体验迁移的策略意图。同时使用 Databricks 和 Power BI 的企业需要立即评估受影响报告范围并制定迁移方案。

3. Snowflake Performance Explorer 可用性升级

来源：Snowflake Release Notes — April 17, 2026

Snowflake 于 4 月 17 日发布 Performance Explorer 的多项可用性改进：新增 Queries/Warehouses/Tables 标签页布局、筛选预设保存、CSV 数据导出、以及侧面板关键字搜索功能。

对数据平台的影响：Performance Explorer 是 Snowflake 用户诊断查询性能和仓库资源利用的核心工具。CSV 导出能力意味着性能数据可以导出到外部分析工具或监控系统，扩展了平台可观测性的数据消费路径。

1. IDC 研究经理王楠：中国数据库市场本土厂商份额历史性反超国际厂商

来源：腾讯云开发者社区 — IDC 研究经理王楠核心观点概要（发布于 2026-04-19，基于腾讯全球数字生态大会演讲）

核心观点：IDC Software Tracker 2024H2 数据显示，中国本地部署关系型数据库市场中，Top 5 本土厂商份额从 2019 年的 22% 上升至 2024 年的 39%，同期 Top 5 国际厂商从 65% 下降至 36%。2024 年中国数据库管理系统软件市场规模 74 亿元，预计 2029 年达 186 亿元（CAGR 20.1%，全球第一）。五大技术方向：AI 融合（向量检索+多模态）、云原生（Serverless）、分布式（存算分离+HTAP）、一体化融合、共享存储集群。

映射到数据平台的判断：份额逆转的意义不仅是信创替代的结果——当本土厂商在核心交易系统中站稳脚跟后，下一阶段竞争焦点将转向 AI 融合能力和海外市场拓展。IDC 的 20.1% CAGR 预测意味着中国数据库市场增速将持续领跑全球。
