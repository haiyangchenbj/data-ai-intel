阿里云日本第五数据中心启用，AI原生数据平台全线进入亚太成熟市场

来源： Alibaba Cloud Press Room
摘要： 6月18日，阿里云宣布东京第五数据中心正式启用，同步将AI原生数据库及数据分析服务全线引入日本市场。核心数据产品包括Data Agent for Analytics（分析洞察）、Meta（数据资产管理）、DAS（数据库智能运维）、DataBridge（多模态数据准备），以及DataWorks Data Agent——可将数据仓库开发周期从天级压缩至分钟级。Model Studio及最新Qwen3.7-Plus模型也随之上线日本区域。
为什么对数据平台重要： 阿里云此次不是简单的IaaS扩容，而是将完整的Agent-native数据平台栈（AI数据库 + 数据开发Agent + 模型平台）首次带入亚太成熟市场。这意味着中国云厂商的数据平台全球化战略已从基础设施层上移到Agent-ready能力层，将直接与AWS、GCP、Azure在日本市场展开数据平台层面的竞争。

Databricks Omnigent Beta 发布：开源多Agent元编排框架，统一编码Agent治理

来源： Databricks Release Notes (June 2026)
摘要： 6月17日（DAIS 2026闭幕前夕），Databricks将Omnigent编码Agent元编排框架推向Beta。Omnigent以Apache 2.0协议开源，作为位于Claude Code、Codex等编码Agent之上的统一元框架，提供组合、控制、协作三层能力：一行配置即可切换Agent运行时、有状态策略替代纯prompt-based管控、支持团队实时共享Agent会话。该框架源自Databricks内部工程团队大规模Agent实践。
为什么对数据平台重要： Omnigent不是又一个Agent工具，而是Agent基础设施层的标准化尝试。当企业同时使用多个编码Agent时，治理碎片化是最大痛点——Omnigent通过在元框架层统一策略执行和预算控制，将数据平台的Agent治理从"每个Agent各自为政"升级到"平台级统一编排"。这与Databricks Unity Catalog的ABAC GRANT策略、Agent Bricks等形成完整的Agent治理栈。

Databricks Genie Spaces 通过 OpenSharing 支持跨组织共享（Beta）

来源： Databricks Release Notes (June 2026)
摘要： 6月17日，Databricks Genie Spaces新增OpenSharing共享能力（Beta），允许供应商将Genie Space的数据资产和指令集以快照形式共享给外部组织。接收方可挂载共享创建本地Genie Space，实现跨组织自然语言数据分析协作。
对数据平台的影响： 这是OpenSharing协议从"共享数据和AI资产"到"共享分析Agent能力"的关键扩展。Genie Spaces成为可通过开放协议分发的Agent产品，意味着数据分析Agent开始具备跨组织可移植性——这对数据平台的Agent生态建设和企业间数据协作模式有直接推动。

Snowflake Data Clean Rooms DCR Agent 进入 Public Preview

来源： Snowflake Release Notes (June 18, 2026)
摘要： 6月18日，Snowflake Data Clean Rooms更新发布，DCR Agent进入Public Preview——用户可通过对话式Snowflake CoWork界面加入协作、运行受众重叠分析等审批模板。同时新增Cortex Code Skills支持以代码方式管理协作生命周期。
对数据平台的影响： Snowflake将Clean Rooms的交互界面Agent化，降低了非技术用户参与数据协作的门槛。DCR Agent本质上是数据治理场景的Agent化落地——数据共享的审批、执行、监控从手动操作转向Agent自动化，这是数据平台治理Agent化的一个具体案例。
