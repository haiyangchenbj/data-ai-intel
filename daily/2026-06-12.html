Top Signals

IBM 与 ServiceNow 扩大合作：watsonx.data 整合 Workflow Data Fabric，打通企业 AI 数据就绪的最后瓶颈

来源： IBM Newsroom
摘要： IBM 和 ServiceNow 于 6 月 11 日宣布扩大战略合作，将 IBM watsonx.data 的数据质量、可观测性和主数据管理能力整合进 ServiceNow Workflow Data Fabric。双方计划在三大领域推出联合方案：应用现代化（IBM Bob + watsonx.data）、企业数据治理（Workflow Data Fabric 扩展）、以及自主基础设施运维（Ansible + Instana + Terraform + Vault 集成到 ServiceNow IT 工作流）。联合方案预计 2026 年下半年可用。
为什么对数据平台重要： 这是企业数据治理层与 IT 工作流平台的深度耦合——watsonx.data 的数据管理能力不再独立存在，而是成为 ServiceNow AI 平台的数据就绪层。对数据平台厂商而言，这意味着数据治理正在从独立平台功能演变为 AI 工作流的前置基础设施，数据治理的采购决策将越来越与 AI 平台选型绑定。

SelectDB 发布"面向 Agent 的实时数据引擎"：CLI+Skill 范式定义 Agent 时代数据基础设施新标准

来源： SelectDB 官网
摘要： 6 月 11 日下午，SelectDB 联合阿里云、阶跃星辰举办"面向 Agent 的实时数据引擎"产品发布会，系统发布 Apache Doris / SelectDB 向 AI Agent 原生演进的技术矩阵。核心发布包括：面向 Agent 的 CLI+Skill 协同设计（将领域工程知识封装为 Agent 可挂载的 Skill，底层工具链重构为 CLI 接口）、多模态统一数据底座（结构化/半结构化/向量/全文检索 SQL 统一查询）、MCP 协议原生支持，以及全链路 Agent 可观测治理平台。
为什么对数据平台重要： 这是国内开源分析数据库领域首个系统化的 Agent-Native 架构宣言。CLI+Skill 范式本质上是在数据库与 AI Agent 之间建立标准化的交互合约，将 Agent 对数据的消费模式从"人机接口适配"升级为"机器原生接口"。如果这一范式被行业采纳，将重塑数据平台的接口层设计和 Agent 可观测性标准。

Product & Tech

Databricks ai_extract 与 ai_classify 正式 GA：SQL 原生 AI 函数嵌入数据平台内核

来源： Databricks June 2026 Release Notes
摘要： Databricks 于 6 月 11 日将 ai_extract 和 ai_classify 两个 AI SQL 函数正式发布为 GA。ai_extract 支持按自定义 schema 从文本和文档中提取结构化数据（嵌套对象、数组、类型验证、置信度评分），ai_classify 支持自定义标签的文本分类（含多标签分类和全局指令）。两者均提供配套的无代码 UI（Information Extraction 和 Classification Agent Bricks）。
对数据平台的影响： AI 能力正在从数据平台的外挂工具下沉为 SQL 原生函数。这种"AI-in-DB"模式意味着数据工程师可以直接在 SQL 查询中调用 AI 能力，无需跳出数据平台调用外部模型服务。对于大规模文档处理、数据分类标注等场景，这将显著降低架构复杂度和工程成本。

Databricks Supervisor Agent 支持 Unity Catalog volumes 作为子代理工具

来源： Databricks June 2026 Release Notes
摘要： Databricks 于 6 月 11 日宣布 Supervisor Agent 现已支持将 Unity Catalog volumes 添加为子代理工具。这意味着在多代理协作架构中，Agent 可以直接访问 Unity Catalog 中托管的文件和数据资产作为工具调用目标。
对数据平台的影响： 这是 Agent 工具链与数据治理层的一次重要集成——Agent 的工具调用权限被纳入 Unity Catalog 的统一治理框架，而非独立于数据平台之外的 Agent 权限体系。对数据平台而言，这意味着"数据治理即 Agent 治理"的趋势正在加速。

OceanBase 发布央国企 AI 一体化数据解决方案：三成央企核心系统已运行其上

来源： TechWeb
摘要： 在 6 月 11 日济南举办的 2026 信息技术应用创新发展大会上，OceanBase 正式发布"央国企 AI 一体化数据解决方案"，并披露全国近 100 家央企中已有 30 余家将核心系统部署于 OceanBase。该方案基于"AI 数据库"构建统一数字底座，支撑经营管控、供应链、资产运维、数据智能等 AI 创新场景，助力央国企从 ERP 向 DRP（数据资源计划）升级。OceanBase 同时入选工信部网安中心"典型解决方案"。
对数据平台的影响： 国产数据库在关键行业核心系统的渗透率正在跨越临界点——从边缘业务系统全面进入央企核心生产系统。AI 转型需求正在加速这一进程，因为国产替代与 AI 就绪正在从并行诉求融合为同一基础设施升级项目的两翼。

【投融资】KKR 联合 Nvidia、Vistra 推出 $100 亿+ Helix AI 数字基础设施平台，前 AWS CEO 出任 CEO

来源： SiliconANGLE
核心数据： 长期资本承诺超 $100 亿，锚定投资方包括 KKR、科威特投资局、Nvidia 和 Vistra

对数据平台的影响： 虽然 Helix 本身是基础设施层项目，但其投资者构成（KKR+主权基金+芯片商+电力公司）和 $100 亿级资本规模标志着 AI 基础设施投资正在从云厂商自建扩展为机构化、平台化的独立赛道。数据平台厂商未来将面对更丰富但也更复杂的底层算力供给选项，算力采购策略需要纳入此类第三方基础设施平台。
