Product & Tech

1. Databricks 实现 SAP BDC 列级治理标签自动同步至 Unity Catalog，并推出治理标签 SQL 管理语句

来源：Azure Databricks Release Notes - April 2026

Databricks 于 4 月 13 日同步发布两项治理能力更新：一是 SAP Business Data Cloud（BDC）的列级个人数据治理标签（fieldSemantics、isPotentiallyPersonal、isPotentiallySensitive）现可自动同步至 Unity Catalog，并可在基于属性的访问控制（ABAC）策略中直接使用；二是 SQL 参考新增 CREATE/ALTER/DROP/DESCRIBE/SHOW GOVERNED TAG 五条治理标签管理语句，治理标签的生命周期管理正式成为 SQL 原语。

对数据平台的影响：这两项更新代表了 Databricks 治理能力的两个方向性进展：向上，Unity Catalog 开始接收外部系统（SAP ERP）的治理语义，数据平台治理从"平台内自治"走向"跨系统策略同步"；向下，治理标签操作从管理界面下沉到 SQL 层面，让数据工程师能用熟悉的方式管理治理规则。对于同时使用 SAP 和 Databricks 的大型企业而言，敏感数据治理策略的统一执行不再需要人工映射。

2. Databricks 与 Lovable 集成：无代码应用平台可直接查询 Lakehouse 数据

来源：Databricks Blog

Databricks 于 4 月 13 日宣布与无代码应用平台 Lovable 集成，推出 Databricks Connector。业务团队可通过自然语言描述需求，由 Lovable 的 AI Agent 直接连接 Databricks 工作区，通过 REST API + OAuth 机器对机器认证查询 Lakehouse 数据，在数小时内构建和部署内部应用。数据始终留在 Databricks 安全边界内，无需 ETL 或数据复制。

对数据平台的影响：数据平台的价值实现长期受限于"最后一公里"——业务团队无法直接消费数据。Lovable 集成代表了一种新的扩展思路：不是让更多人学 SQL，而是让应用构建平台直接接入 Lakehouse。这降低了数据消费端的技术门槛，同时通过 OAuth + Unity Catalog 治理保持数据安全边界。对于 Databricks 而言，这是从"数据团队工具"向"全公司数据基础设施"定位演进的又一步。

3. Databricks Mosaic AI Vector Search 推出内置检索质量评估功能（Beta）

来源：Azure Databricks Release Notes - April 2026

Databricks 于 4 月 13 日发布 Mosaic AI Vector Search 的内置检索质量评估（Retrieval Quality Evaluation）功能，进入 Beta 阶段。该功能可衡量和比较不同搜索策略在用户自有数据上的相关性表现，帮助开发者选择最优检索方案。

对数据平台的影响：RAG 应用的质量瓶颈往往在检索环节而非生成环节。将检索质量评估内置到向量搜索服务中，降低了 RAG 应用调优的门槛，是 Databricks 在 AI 应用基础设施层面持续补全工具链的体现。
