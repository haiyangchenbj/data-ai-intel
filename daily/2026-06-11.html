Databricks 将 Delta Sharing 捐赠给 Linux Foundation，升级为 OpenSharing 开放标准

来源：Databricks Press Release / Linux Foundation PR Newswire
摘要： Databricks 于 6 月 10 日宣布将 Delta Sharing 协议升级为 OpenSharing，并正式捐赠给 Linux Foundation 作为独立开源项目。OpenSharing 是首个覆盖 Agent Skills、AI 模型和非结构化数据的开放共享协议，同时新增了对 Apache Iceberg IRC 客户端的支持，使 Iceberg 生态用户可以直接消费共享资产。同日，Databricks 还宣布与 MinIO（已 GA）、Everpure、Qumulo、VAST Data 等存储厂商建立 OpenSharing 存储生态，使本地部署数据无需迁移即可通过 Unity Catalog 安全访问。
为什么对数据平台重要： Delta Sharing 此前是 Databricks 主导的开源协议，捐赠给 Linux Foundation 意味着数据与 AI 资产共享标准正式进入基金会治理时代。OpenSharing 是数据平台领域首个从"表格式互操作"扩展到"全 AI 栈资产互操作"的协议——不只是共享数据表，而是共享 Agent 技能、AI 模型和非结构化数据。此举直接回应了 Agent 时代跨平台协作的核心需求，同时通过支持 Iceberg IRC 客户端消解了 Delta Lake 和 Iceberg 生态之间的共享壁垒。存储生态的搭建则解决了一个长期痛点：本地数据上云的迁移成本与合规风险。

Zilliz 发布 Vector Lakebase，向量数据库升级为统一数据平台

来源：PR Newswire
摘要： 全球最广泛采用的向量数据库 Milvus 母公司 Zilliz 于 6 月 10 日发布 Vector Lakebase Public Preview——将 Zilliz Cloud 从向量检索引擎升级为统一数据平台。Vector Lakebase 基于开源列式格式 Vortex 构建共享湖原生存储层，在单一数据底座上支持五大能力：分层实时检索（性能优先/容量优化/分层存储三档）、按需搜索（成本仅为 Serverless 方案的 1/15）、外部数据湖搜索（零拷贝支持 Iceberg/Parquet/Lance/Vortex）、全频谱 AI 搜索（向量+全文+JSON+地理空间混合检索）以及湖原生存储统一管理。
为什么对数据平台重要： 这是向量数据库品类首次系统性向统一数据平台演进。Vector Lakebase 的核心命题——"同一份向量数据驱动生产查询、交互式发现和批量分析，无需拷贝或迁移"——直接回应了 AI 数据闭环的最大痛点：数据在检索、探索和训练三个环节之间反复拷贝带来的成本和延迟。在架构层面，基于 Vortex 的湖原生存储意味着向量数据和结构化数据可以在同一套开放格式上统一管理，这对 Agent 记忆系统和 RAG 管线的数据基础设施设计有直接影响。

【预告】SelectDB 联合阿里云、阶跃星辰发布"面向 Agent 的实时数据引擎"

来源：SegmentFault
为什么值得继续看： SelectDB（Apache Doris 商业化公司）将于 6 月 11 日 14:00 举办线上发布会，主题为"面向 Agent 的实时数据引擎"。发布会将首次解构 SelectDB/Doris 如何向 AI Agent 完成原生演进，并推出专为 Agent 时代打造的数据技术矩阵，覆盖多模态原生融合底座、数据工具链和全链路可观测治理平台。阿里云和阶跃星辰作为生态伙伴参与，暗示产品可能涉及云原生集成和大模型对接能力。
需要等待什么信号确认： 发布会实际产品能力细节，特别是"Doris 面向 Agent 原生演进"的技术实现路径（是否包含 Agent Memory 引擎、多模态存储、实时推理集成等），以及生态伙伴合作深度（阿里云集成方式、阶跃星辰模型融合方案）。

【降级】Upriver 获 $14M 种子轮，AI Agent 自主管理企业数据管道

来源：Newswire
为什么值得继续看： Upriver 构建了一个 AI-native 数据工程平台，通过 Agent 系统自动接管企业数据管道的质量检测、问题修复和持续维护，已与 Databricks 和 Snowflake 建立正式合作。Gartner 数据显示 38% 的技术领导者将数据质量问题列为 AI 项目失败的直接原因，Upriver 的 agentic 数据工程路线直接瞄准这一痛点。种子轮由 Valley Capital Partners 和 Hetz Ventures 领投，天使投资人包括 New Relic 创始人 Lew Cirne 和 Cyera 创始人。
需要等待什么信号确认： 产品 GA 时间线和早期客户的实际采用效果（当前客户包括 Unity 和 DMGT）。Agentic 数据工程的可靠性验证——特别是在生产环境中自主修复数据管道是否会产生级联故障。

【跟踪】Databricks Workspace 权限模型即将结构性变更

来源：Databricks What's Coming
为什么值得继续看： Databricks 宣布将变更 Workspace 权限继承机制：从 6 月 15 日起开放 opt-in，9 月 14 日强制执行。新模型下新增用户不再自动继承 `users` 系统组的 Workspace Access 和 SQL Access 权限，需显式授权。这一变更将直接影响所有使用 Terraform、SCIM API 或自定义脚本管理 Databricks 权限的企业，且 `users` 和 `admins` 系统组不再支持嵌套。
需要等待什么信号确认： Opt-in 开启后企业迁移的实际反馈，特别是自动化权限管理脚本的兼容性问题和 SCIM 同步对迁移克隆组的处理。这可能推动 Unity Catalog 权限管理的标准化实践。
