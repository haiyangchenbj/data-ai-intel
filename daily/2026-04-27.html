Product & Tech

1. Databricks Workspace Base Environments 正式 GA

来源：Databricks Release Notes — April 2026
Databricks 于 04-24 宣布 Workspace Base Environments 正式 GA。该功能允许工作区管理员为无服务器笔记本创建和管理预构建、缓存的 Python 环境，包括预安装的库和配置，消除用户每次启动笔记本时的环境初始化等待。启用合规安全配置文件的工作区将在 2026 年 5 月下旬开始推出。
对数据平台的影响：无服务器计算的"冷启动"问题一直是企业采用的痛点之一。预构建环境 GA 意味着 Databricks 在无服务器笔记本的开发者体验上补齐了关键一环——管理员可以标准化团队的依赖环境，开发者获得秒级启动体验。这是无服务器 notebook 走向企业级生产就绪的必要基础设施。

2. Databricks Supervisor Agent SDK 进入 Beta——多 Agent 编排走向编程化

来源：Databricks Release Notes — April 24, 2026
Databricks 于 04-24 发布 Supervisor Agent 的 Python SDK 编程接口（Beta），允许开发者通过 Databricks SDK 以代码方式创建、配置和管理 Supervisor Agent 及其下属工具，用于构建协调式多智能体系统。此前 Supervisor Agent 仅支持通过 UI 配置。
对数据平台的影响：多 Agent 系统从 UI 配置走向 SDK 编程管理，标志着 Agent 编排进入工程化阶段。对数据平台而言，这意味着 Agent 不再是孤立的"聊天助手"，而是可以被纳入 CI/CD 流水线、版本管控和自动化测试的生产组件——与传统数据管道的管理方式趋同。

Views & Research

1. Futurum Group：Google Agentic Data Cloud 将成为 Agentic AI 的"语义核心"，但"智能锁定"风险值得警惕

来源：Futurum Group — Going Beyond the Data Graveyard With Google's Agentic Data Cloud
Futurum 分析师 Brad Shimmin 在 04-24 发表深度分析，认为 Google 的 Knowledge Catalog 不是简单的元数据仓库重命名，而是一个"动态语义引擎"——它将对话从"数据物理位置"转向"数据业务含义"。同时 Smart Storage（GCS 原生能力）在文件落地时自动标记、生成向量嵌入、提取实体，有效"点亮"企业暗数据。但 Shimmin 明确指出两大风险：跨云出口费用仍是硬约束，更重要的是"智能锁定"——语义映射和 Agent 逻辑深度绑定 Gemini 模型，"迁移大脑比迁移数据困难得多"。
映射到数据平台的判断：这是首个明确区分"存储锁定"和"语义锁定"的主流分析师判断。随着数据平台竞争从存算层上移到上下文与推理层，企业的迁移壁垒不再是数据搬迁成本，而是重建语义映射和 Agent 工作流的成本。这对多云数据架构的选型逻辑有根本影响。
