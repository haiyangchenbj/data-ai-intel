Top Signals

1. Google Cloud × Collibra 深化合作：双向元数据集成进入 Public Preview

来源：PR Newswire 官方新闻稿 / Collibra 官方新闻

Google Cloud 与 Collibra 于 04-22 宣布扩展战略合作伙伴关系，聚焦开放湖仓环境中的数据治理。核心发布是 Collibra 与 Google Cloud Dataplex 的双向集成（Public Preview）：出站方向——Collibra 中定义的业务上下文、数据血缘、质量和治理策略原生流入 Dataplex；入站方向——Dataplex 的技术元数据和发现洞察自动回传至 Collibra，确保企业记录系统实时更新。联合客户在 Google Cloud Knowledge Catalog 中即可获得统一的数据发现、语义和合规能力。

为什么对数据平台重要：这是主流云平台首次与独立治理厂商实现生产级双向元数据同步。此前各平台的治理能力（Unity Catalog / Horizon / Dataplex）均以内建为主，Collibra 的跨平台定位意味着治理上下文不再被锁定在单一平台内。对多云/开放湖仓客户而言，业务含义可以跟随数据跨平台流动，这改变了治理从「平台附属品」到「独立基础设施层」的定位。

Product & Tech

1. Databricks Lakeflow Designer 进入 Public Preview：可视化无代码数据准备 + AI 原生

来源：Databricks Release Notes - April 2026 / Lakeflow Designer 文档

Databricks 于 04-22 正式将 Lakeflow Designer 推入 Public Preview。这是一个面向分析师的可视化画布工具，支持拖放式 DAG 编排（Filter / Join / Aggregate / Reshape 等算子）、Excel/CSV 文件直接拖入画布、逐步预览中间结果，以及关键差异化——内置 Genie Code 自然语言驱动的转换生成，所有交互均为 Agentic 模式并利用 Databricks 平台上下文。所有可视化转换有底层代码支撑，可迁移至生产环境。

对数据平台的影响：Lakeflow Designer 补齐了 Databricks 在「非工程师用户」群体的数据准备入口。与 Microsoft Fabric Dataflows、Informatica IDMC 等竞品不同，Designer 从设计之初就嵌入 AI（Genie Code），将自然语言作为一等公民接口。这意味着数据准备的门槛从「会写 SQL/Python」降到「会描述需求」，平台用户基数有望进一步向业务分析师扩展。
