DAIS 2026 Day 1——Agent 可观测性正式进入数据平台治理栈

来源： Monte Carlo GlobeNewswire
摘要： 在 Databricks Data + AI Summit 2026 开幕日，Monte Carlo 宣布成为 Agent Bricks 的首个可观测性启动合作伙伴。该集成将 Monte Carlo 的监控能力从 Delta Lake 表质量和 Lakeflow 管道健康，扩展到 Agent Bricks 层——覆盖 Agent 的工具调用、检索步骤、模型交互、编排工作流及数据输入，提供从原始数据到 Agent 决策的连续审计链路。
为什么对数据平台重要： 这是数据平台治理栈首次将 Agent 行为层纳入统一的端到端可观测性框架。此前 Agent 在数据平台上是"信任黑箱"——数据团队能监控表的新鲜度、schema 漂移和管道异常，但无法追溯到某个 Agent 的错误推理是否源自数据层、模型层还是编排层。Monte Carlo 三层联动的可观测性（Delta Lake → Lakeflow → Agent Bricks）补齐了这个缺失的治理维度，使 Agent 生产部署具备与数据管道同等级别的运维可信度。

Imply Lumi Loglake——湖仓架构扩展到非结构化日志搜索

来源： Imply Business Wire
摘要： Imply 在 DAIS 2026 期间发布 Lumi Loglake，支持在 AWS S3、Delta Lake 和 Apache Iceberg 等对象存储上直接对非结构化日志进行交互式搜索，无需预先定义 schema、建立数据目录或执行 rehydration 工作流。该方案基于 Apache Druid 引擎，将存算分离范式应用于非结构化可观测性数据，声称可降低软件成本 70%+、硬件成本 40%+。
为什么对数据平台重要： Lumi Loglake 将湖仓存算分离范式首次系统性地推进到非结构化日志领域。可观测性和 SIEM 数据是 AI 时代增长最快的数据类别之一，传统方案要求先索引后查询，成本随数据量线性增长。Loglake 的"先搜索、后优化"模式让企业可以按查询量付费而非按存储量付费，对数据平台架构的启示在于：湖仓范式不应局限于结构化分析数据，非结构化可观测性数据同样是存算分离的天然适用场景。

Databricks ai_query 正式 GA——通用 AI 函数使 SQL/Python 原生调用模型

来源： Databricks June 2026 Release Notes
摘要： Databricks ai_query 函数于 6 月 15 日进入正式可用（GA）。该函数允许用户直接从 SQL 或 Python 中调用任何支持的 AI 模型，在湖仓环境中实现数据查询与 AI 推理的无缝融合，无需额外的 API 集成或中间层服务。
对数据平台的影响： ai_query GA 标志着数据平台中的 AI 调用能力正在从外挂工具收敛为内置 SQL 原语。分析师和工程师可以在同一个查询中混合传统聚合与 LLM 推理，极大降低了 AI 能力嵌入数据管道的工程门槛。这种"AI 函数化"趋势将改变数据平台的产品形态——AI 不再是一个独立模块，而是 SQL 层的一个内置函数族。

Databricks Genie One 多项能力 GA + Genie Mobile 进入 Public Preview

来源： Azure Databricks AI/BI Release Notes
摘要： 6 月 15 日，Databricks Genie One 的 Chat 和 Documents 功能正式 GA——Chat 提供跨 Genie Spaces、仪表盘、查询和指标视图的统一自然语言问答界面；Documents 允许用户在对话中直接创建和分享文档。同日，Genie Mobile 应用进入 Public Preview，支持 iOS 和 Android。
对数据平台的影响： Genie One Chat GA 标志着 Databricks 的自然语言分析能力从实验阶段进入生产就绪。Chat 将 BI 查询、仪表盘和文档整合在同一对话界面，降低了非技术用户的数据消费门槛。Mobile Preview 则首次将湖仓分析能力延伸到移动端，对需要随时访问数据的高管和一线运营人员具有直接实用价值。

Databricks Lakebase Postgres 登陆 GCP——Beta 版覆盖三大区域

来源： Databricks GCP June 2026 Release Notes
摘要： Databricks 于 6 月 15 日将 Lakebase Autoscaling（Serverless Postgres 数据库）的 Beta 版扩展至 Google Cloud Platform，首批支持 us-east4、us-central1 和 europe-west3 三个区域。Lakebase 提供自动扩缩容、缩容至零和数据库分支等能力。
对数据平台的影响： Lakebase 在多云扩展上的进展——此前已支持 AWS 和 Azure，GCP Beta 的上线意味着 Databricks 的操作型数据库层即将覆盖三大云平台。对于采用 GCP 的企业而言，Lakebase 的 Postgres 兼容性意味着可以在湖仓统一治理框架下运行事务型工作负载，减少对独立 RDS/Cloud SQL 的依赖。
